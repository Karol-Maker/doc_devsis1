# 3\. Principais Recursos e Funcionalidades

O Sistema de Gestão do Vinhedo Parreiral foi projetado para integrar operações administrativas, vendas, estoque, eventos e serviços turísticos. Ele conta com módulos específicos para atender às necessidades dos administradores do vinhedo, funcionários e clientes que acessam o portal. Abaixo estão descritas as principais funcionalidades.

## Módulos do Sistema

### 1\. Cadastro de Clientes

Permite que os administradores registrem e gerenciem informações e histórico completos dos clientes, incluindo nome, telefone, e-mail, endereço, preferências de compra, histórico de visitas e identificador único.

### 2\. Cadastro de Produtos e Vinhos

Os administradores podem cadastrar vinhos, eventos e produtos derivados da uva e serviços turísticos.Cada item contém dados como safra, tipo de uva, lote, preço, descrição e quantidade disponível.

### 3\. Controle de Estoque

O sistema atualiza automaticamente o estoque sempre que uma compra é realizada ou quando um item é reservado. Também envia alertas para produtos com baixo volume, safras esgotando ou eventos com vagas limitadas.

### 4\. Vendas e Emissão de Nota Fiscal

Clientes podem comprar vinhos e agendar experiências diretamente no sistema. Após a confirmação da compra, o sistema gera nota fiscal, atualiza o estoque e registra a transação no histórico do cliente.

### 5\. Pré-reservas de Produtos

Se um vinho ou produto estiver indisponível, o cliente pode realizar uma pré-reserva e receber prioridade quando a nova safra ou lote estiver disponível.

### 6\. Histórico de Compras, Reservas e Pagamentos

Cada cliente tem acesso ao seu histórico completo de compras, reservas de eventos, pagamentos realizados e saldo devedor (quando houver).

### 7\. Promoções e Benefícios Sazonais

Administradores podem cadastrar promoções, descontos, cupons e benefícios especiais (como degustação gratuita para clientes fidelizados). Os clientes podem visualizar e aplicar esses benefícios durante compras.

### 8\. Gerenciamento de Eventos e Experiências

Administradores podem criar eventos como degustações, visitas guiadas, cursos, jantares harmonizados e festivais. Clientes podem visualizar a agenda, reservar vagas, cancelar ou reagendar atividades.

### 9\. Portal do Cliente

#### Área dedicada onde o cliente pode:

* Atualizar seus dados pessoais;

* Consultar compras;

* Gerenciar reservas;

* Acessar promoções e pré-vendas;

* Visualizar seus eventos confirmados.

### 10\. Relatórios Gerenciais

#### O sistema gera relatórios sobre:

* Vendas e faturamento;

* Inadimplência;

* Participação em eventos;

* Produtos mais procurados e vendidos;

* Desempenho de promoções.

3.2 Requisitos Não Funcionais

| RNF01 – Desempenho | O sistema deve carregar páginas e realizar operações em até 3 segundos. |
| :---- | :---- |
| RNF02 – Segurança | Dados dos clientes e administradores devem ser protegidos com autenticação, criptografia e controle de acesso. |
| RNF03 – Disponibilidade | O sistema deve operar no mínimo 99% do tempo mensal. |
| RNF04 – Usabilidade | A interface deve ser simples, intuitiva e adequada para público geral e funcionários. |
| RNF05 – Confiabilidade | O sistema deve registrar transações sem risco de perdas de dados. |
| RNF06 – Escalabilidade | Deve suportar aumento de usuários, produtos e eventos sem perder desempenho. |
| RNF07 – Compatibilidade | Funcionará em navegadores modernos e dispositivos diversos (desktop, tablet, mobile). |
| RNF08 – Manutenibilidade | O código deve ser modular, organizado e bem documentado. |
| RNF09 – Privacidade | Dados pessoais devem seguir normas da LGPD. |
| RNF10 – Backup | O sistema deve realizar backups automáticos periódicos para evitar perda de dados. |
| RNF11 – Integridade | As informações devem ser consistentes mesmo em caso de erro ou queda de energia. |
